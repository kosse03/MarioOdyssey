# 09. 콘텐츠·설정·빌드 구조

## 1. 프로젝트 파일

`MarioOdyssey.uproject` 기준:

- EngineAssociation: 5.4
- Runtime 모듈: `MarioOdyssey`
- 대상 플랫폼: Windows
- `.uproject` 명시 플러그인: ModelingToolsEditorMode, SkeletalMeshModelingTools
- 프로젝트 로컬 개발 플러그인: `Plugins/Developer/RiderLink` (Editor 빌드에서 함께 컴파일됨)

명시된 두 플러그인은 모델링/에디터 작업 성격이고 RiderLink는 IDE 연동용 개발 플러그인이다. 핵심 런타임 시스템은 Engine 모듈과 프로젝트 C++로 구성된다.

## 2. Build.cs 의존성

| 모듈 | 사용 목적 |
|---|---|
| Core, CoreUObject, Engine | UE 기본 객체/Actor/Component |
| InputCore, EnhancedInput | InputAction과 MappingContext |
| NavigationSystem, AIModule | 굼바 AI 순찰/복귀, 해제 위치 투영 |
| LevelSequence, MovieScene | 시작·Arena·보스 컷신 |
| UMG, Slate, SlateCore | HUD와 키 입력 Widget |

단일 모듈이라 Public/Private 폴더는 나뉘어도 기능별 별도 Module 경계는 없다.

## 3. 맵 구성

프로젝트 전체 `.umap`은 7개다.

### 실제 게임 맵

| 경로 | 용도 |
|---|---|
| `/Game/Maps/Map` | 기본 시작/에디터 맵, 탐색과 슈퍼문 수집 |
| `/Game/Maps/Arena` | 포탈 목적지, 인트로와 보스전 |
| `/Game/Maps/ThirdPersonMap` | 템플릿/테스트 잔존 가능성 |

### 템플릿/Starter 맵

- `/Game/StarterContent/Maps/Advanced_Lighting`
- `/Game/StarterContent/Maps/Minimal_Default`
- `/Game/StarterContent/Maps/StarterMap`
- `/Game/ThirdPerson/Maps/ThirdPersonMap`

Packaging의 `MapsToCook`에는 Map과 Arena만 지정돼 있다. 게임 실행 경로에는 맞지만 테스트 맵은 패키지에 포함되지 않을 수 있다.

## 6. 입력 자산

`Content/Input`에는 다음 7개 자산이 있다.

- `IMC_Player`
- `IA_Move`
- `IA_Look`
- `IA_Jump`
- `IA_Crouch`
- `IA_Run`
- `IA_ThrowCap`

DefaultInput은 EnhancedPlayerInput/EnhancedInputComponent를 기본 클래스로 지정한다. 실제 키 매핑은 바이너리 `IMC_Player.uasset`에 있어 C++ 정적 분석만으로 모든 키를 확정할 수 없다. 시작 화면의 C 키는 별도 Widget 코드에 명시돼 있다.

## 7. 렌더링 설정

| 설정 | 값 |
|---|---|
| RHI | DX12 |
| Shader Model | PCD3D_SM5, PCD3D_SM6 |
| Dynamic GI | Lumen |
| Reflection | Lumen |
| Shadow | Virtual Shadow Map |
| 타깃 하드웨어 | Desktop, Maximum Quality |

고품질 데스크톱 렌더링을 전제로 한다. 저사양/모바일 Scalability와 별도 Target은 현재 구성에서 보이지 않는다.

## 8. 커스텀 충돌 채널

| 채널 | GameTraceChannel | 기본 응답 | 목적 |
|---|---:|---|---|
| Monster | 1 | Block | 몬스터 캡슐/보스 부위 |

| Detector | 3 | Ignore | 벽/난간/접촉/카운터 감지 |

## 9. 커스텀 충돌 프로필

| 프로필 | 핵심 역할 |
|---|---|
| `Player_Capsule` | 플레이어 물리 캡슐 |
| `Player_Mesh` | 시각 Mesh, 대부분 Ignore |
| `Monster_Capsule` | Monster ObjectType, 캐피/Detector Overlap |
| `Monster_Mesh` | 몬스터 시각 Mesh |
| `Monster_ContactSphere` | Pawn/Monster 접촉 감지 |
| `Detector_Wall` | WorldStatic/Dynamic 벽 Overlap |
| `Detector_Ledge` | 난간 후보 Overlap |
| `Cap_Flight` | Monster Overlap, 월드 Block |
| `Wall_Climbable` | Detector Overlap 허용 |
| `Wall_Normal` | 일반 벽 |
| `Boss_Fist_Pass` | 평상시 손, Pawn Ignore |
| `Boss_Fist_Attack` | 공격 중 Pawn Overlap |
| `Boss_Head_CounterOnly` | 마리오/캡처 손 머리 판정 |
| `Boss_IceShard_Rain` | 마리오 피해용 낙하 얼음 |
| `Boss_IceShard_Counter` | 캡처 손 Detector용 카운터 얼음 |
| `Boss_IceTile_Hazard` | 얼음 타일 Overlap |

프로필 이름은 C++의 `SetCollisionProfileName`과 Project Settings가 문자열로 결합되는 API 계약이다. 이름 변경 시 컴파일 오류 없이 런타임 판정만 깨질 수 있어 자동 검증이 필요하다.

## 10. 시작 클래스와 하드 참조

C++ 생성자에서 다음 Blueprint 경로를 직접 하드 참조한다.

| 클래스 | 경로 |
|---|---|
| GameMode | `/Game/_BP/Character/Mario/BP_Mario` |
| MarioPlayerController | `/Game/_BP/UI/WBP_HUD` |
| MarioStartFlowActor | `/Game/_BP/UI/WBP_StartScreen` |

경로가 바뀌면 로드 실패한다. GameMode만 C++ Mario로 fallback하고, HUD/StartWidget은 클래스가 없으면 해당 UI를 생성하지 않는다.

## 11. 패키징 설정

- Development 구성
- Pak + IoStore 사용
- Oodle Kraken 압축
- Map/Arena만 명시 Cook
- Prerequisites 포함
- InternationalizationPreset과 Stage Culture는 English
- Crash Reporter 미포함

프로젝트 내 한국어 주석은 런타임 로컬라이징과 무관하다. 게임 UI에 한국어 텍스트가 있다면 현재 Stage Culture 설정과 Localization Target을 별도로 점검해야 한다.

## 12. 템플릿 잔여와 정리 후보

- `DefaultGame.ini`의 ProjectName이 아직 `Third Person Game Template`
- StarterContent 자동 추가 설정 유지
- 템플릿 `MarioOdysseyCharacter` 클래스가 실제 `AMarioCharacter`와 별도로 잔존
- ThirdPersonMap이 두 경로에 존재
- `_BP/Character` 아래 실험성 이름의 중복 굼바 자산이 보임
- `Content/Data`는 비어 있음

이 항목들은 즉시 버그는 아니지만 패키지 크기, 신규 개발자 혼동, 리다이렉터/참조 관리 비용을 높인다. Reference Viewer와 Size Map으로 실제 사용 여부를 확인한 뒤 정리해야 한다.
