## 먼저 읽을 결론

프로젝트의 중심은 `AMarioCharacter`의 복합 이동과 `UCaptureComponent`의 Pawn 전환이다. 수집 진행도는 `UMarioGameInstance`의 이벤트 허브를 통해 HUD, 포탈, 이동 플랫폼에 전달된다. 후반부는 캡처 기믹 자체를 보스 공략 수단으로 확장한다.

현재 가장 중요한 코드 리스크는 캡처된 Pawn의 피해가 “마리오 HP로 전달된다”는 주석과 달리 실제로는 피격 리액션만 호출하고 HP를 차감하지 않는 점이다. 세이브도 디스크 저장이 아닌 GameInstance 세션 유지까지만 구현돼 있다. 자세한 근거와 검증법은 10번 문서에 정리했다.
