# H1. 복합 이동 상태 조정

## 1. 왜 어려운가

마리오 이동은 하나의 상태기가 아니다. CharacterMovement의 MovementMode 위에 Roll, GroundPound, Dive, WallAction, JumpChain, Crouch, DownhillBoost가 각각 자체 상태와 Timer를 가진다.

어려운 지점은 기술 구현 자체보다 “한 기술이 다른 기술의 물리 값을 바꾼 뒤 어떤 종료 경로에서도 정확히 원복되는가”다.

## 2. 상태 축

| 축 | 대표 상태 | 바꾸는 엔진 값 |
|---|---|---|
| MovementMode | Walking, Falling, None | 지상/공중/정지 판정 |
| Roll | Start, Loop, End | MaxWalkSpeed, Friction, Braking |
| GroundPound | Preparing, Pounding, Stunned | GravityScale, Velocity, 입력 |
| Dive | bool/AirAction | AirControl, Acceleration, 회전 정책 |
| WallAction | SlideStart, SlideLoop, WallKick | GravityScale, Z Velocity, 입력 |
| Crouch | Held/Crouched | Capsule 높이, 카메라 Offset, 속도 |
| Downhill | Alpha/Hold | MaxWalkSpeed |
| Damage/Death | HitStun/InputLock/GameOver | 입력, MovementMode, Montage |
| Capture | Component 상태 | 위 모든 액션 강제 종료 |

같은 속성에 여러 시스템이 쓰는 것이 핵심 충돌원이다. 예를 들어 `MaxWalkSpeed`는 기본 이동, Roll, DownhillBoost가 모두 수정한다. `GravityScale`은 GroundPound, Dive 복구, WallSlide, Capture 정리가 만진다.

## 3. 입력 우선순위

### Jump

```text
InputLocked면 종료
→ Rolling이면 Roll Abort
→ Wall Slide면 WallKick 실행 후 종료
→ WallKick 상태면 종료
→ Falling이면 종료
→ GroundPound/PoundJump 창 처리
→ Crouch 파생 점프 시도
→ LongJump/Backflip이면 종료
→ 일반 3단 Jump
```

벽차기가 일반 점프보다 먼저고, 파운드 반동 점프가 웅크리기 파생 점프보다 먼저다. 이 순서가 바뀌면 같은 입력에서 다른 기술이 나간다.

### Crouch

```text
지상 → Crouch Hold
공중 + GroundPound 진행 중 → Dive
그 외 공중 → GroundPound 시작
```

### Run

```text
웅크린 지상 + Roll 가능 → Roll
그 외 → Running 활성화
```

## 4. 액션 호환 표

| 현재 액션 | 새 Roll | 새 GroundPound | 새 Dive | WallSlide | 일반 이동 |
|---|---:|---:|---:|---:|---:|
| Roll | - | 금지/공중 이탈 시 Abort | 금지 | 금지 | Roll 내부 조향만 |
| LongJump | 지상 착지 후 가능 | 금지 | 직접 진입 없음 | 명시 금지 | 공중 제어 |
| Backflip | 착지 후 가능 | 가능 | Pound 경유 | 가능 후보 | 공중 제어 |
| GroundPound Prepare | 금지 | - | 허용 | 금지 | 차단 |
| GroundPound Pounding | 금지 | - | 허용 | 금지 | 차단 |
| Dive | 금지 | 금지 | - | 금지 | 차단 |
| WallSlide | 금지 | 금지 | 금지 | - | 차단 |
| PoundJump | 착지 후 가능 | 공중 1회 규칙 | Pound 경유 가능 | 가능 후보 | 공중 제어 |

## 5. 안전한 원복 불변식

어떤 액션을 끝내든 일반 상태에서 다음이 맞아야 한다.

```text
GravityScale == DefaultGravityScale
AirControl == DefaultAirControl
MaxAcceleration == DefaultMaxAcceleration
BrakingDecelerationFalling == DefaultBrakingDecelFalling
bOrientRotationToMovement == bDefaultOrientRotationToMovement
GroundFriction == DefaultGroundFriction
BrakingDecelerationWalking == DefaultBrakingDecelWalking
BrakingFrictionFactor == DefaultBrakingFrictionFactor
MaxWalkSpeed == 현재 Walk/Run/Crouch 기준값 (+ 유효한 DownhillBoost만)
```

현재 코드는 `AbortRoll`, `EndDive`, `ResetWallSlide`, `OnCaptureBegin/End` 등이 각각 일부 또는 전체를 복원한다. 신규 액션이 이 값을 추가로 변경하면 모든 비정상 종료 경로를 다시 검토해야 한다.

## 6. 캡처가 전역 Reset 지점인 이유

캡처는 PlayerController가 다른 Pawn으로 이동하는 경계다. 기존 마리오 액션이 남으면 숨겨진 마리오의 Timer가 끝난 뒤 MovementMode나 카메라 값을 바꿀 수 있다.

그래서 `OnCaptureBegin`은 다음을 한 번에 정리한다.

- 모든 Montage 중지
- HitStun과 InputLock 제거
- Roll Abort, Dive End
- GroundPound와 PoundJump Timer/flag 종료
- Wall Timer/normal/grace 종료
- Jump chain 초기화
- Downhill boost 제거
- Crouch/Run 해제
- `FMarioState{}` 재초기화
- CharacterMovement 전체 기본값 복구

`OnCaptureEnd`도 대칭적으로 다시 정리한다. 진입 중 호출 순서나 몬스터 파생 코드가 마리오 값을 바꿔도 복귀 기준을 확보하기 위한 이중 방어다.

## 7. 개선 설계안

기능을 유지하며 안정성을 높이려면 다음 순서가 적합하다.

1. 엔진 물리 기본값을 `FMarioMovementDefaults` 한 구조체로 캡처
2. 액션마다 `Enter/Exit(Reason)` 한 쌍만 허용
3. 각 액션의 활성 여부를 enum에서 파생하고 중복 bool 제거
4. `RequestAction(NewAction)`이 호환 표를 검사해 기존 액션 종료 후 진입
5. Capture/Damage/Death는 `CancelAllActions(Reason)`만 호출
6. Development 빌드에서 불변식 `ensure` 추가

이 프로젝트에서 가장 회귀 비용이 큰 곳이므로 새 이동기를 추가하기 전에 이 정리가 선행되는 편이 안전하다.
